# Design Principles

## Direction

- Build a premium portfolio, not a generic landing page.
- Lead with proof, craft, and personality.
- Let visuals carry the experience where possible.
- Keep copy sharp and specific.
- Avoid decorative noise that does not support the work.

## UX Principles

- First viewport must clearly communicate who the client is and why their work matters.
- Projects should be easy to scan, compare, and open.
- Contact intent should remain obvious without feeling pushy.
- Navigation should stay lightweight.
- Responsive behavior should feel designed, not merely stacked.

## Engineering Principles

- Build reusable sections and components.
- Keep content/data separate from presentation where practical.
- Use design tokens instead of one-off values.
- Avoid hardcoded layout hacks.
- Verify desktop and mobile visually before delivery.
- Keep React code senior-level but plain: descriptive names, small components, no unnecessary abstractions.
- Use animations to support interaction and storytelling, not to decorate every element.

## Accessibility Principles

- Maintain strong text contrast.
- Use semantic structure for sections and headings.
- Support keyboard navigation.
- Avoid motion that blocks comprehension.
- Provide meaningful alt text for work images.
- Respect `prefers-reduced-motion`.
